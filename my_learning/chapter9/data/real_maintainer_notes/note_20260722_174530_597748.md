## 二、GSSC 处理流程

GSSC 是四个阶段的缩写，在代码中对应以下方法：

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│   Gather    │ →  │   Select    │ →  │  Structure  │ →  │  Compress   │
│  (收集)     │    │  (筛选)     │    │  (整理)     │    │  (压缩)     │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
      ↓                  ↓                  ↓                  ↓
  def _gather()    def _select()    def _structure()   def _compress()
  (第131行)        (第395行)        (第556行)          (第770行)
```

### 各阶段功能

| 阶段 | 方法位置 | 功能描述 |
|------|----------|----------|
| **Gather** | `builder.py:131` | 收集候选信息（如代码片段、文件内容） |
| **Select** | `builder.py:395` | 筛选高价值信息，去除冗余 |
| **Structure** | `builder.py:556` | 整理成固定结构（如 JSON、Markdown） |
| **Compress** | `builder.py:770` | 超出预算时进行压缩（如摘要、截断） |
| **Build** | `builder.py:901` | 统一入口，串联上述四个阶段 |

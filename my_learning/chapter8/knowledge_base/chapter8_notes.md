# Chapter 8 学习笔记

智能体记忆系统用于保存和检索过去的交互信息。

工作记忆用于保存当前任务需要的信息。工作记忆容量有限，
并且其中的信息通常具有生存时间。当信息超过生存时间后，
系统可以自动将其清理。重要的工作记忆可以被整合到长期记忆中。

情景记忆用于保存具体发生过的事件，例如用户在某个时间提出了
什么问题，智能体执行了什么操作，以及最终得到了什么结果。
情景记忆通常需要保存时间、会话编号和事件类型等信息。

语义记忆用于保存比较稳定的事实、概念和知识。例如，用户正在
学习 Python Agent 开发，或者 RAG 是检索增强生成的缩写。
语义记忆强调知识本身，而不是知识在什么时候产生。

记忆管理器负责统一协调不同类型的记忆。它可以接收添加、
搜索、更新、整合和遗忘等操作，并将操作转发给对应的记忆模块。

RAG 是检索增强生成。它会在大语言模型回答问题之前，
先从外部知识库中检索与问题相关的内容，再将检索内容加入
提示词，帮助大语言模型生成更加准确、更加可靠的回答。

RAG 的数据准备阶段通常包括文档读取、文本分块、向量化和存储。
查询阶段通常包括问题向量化、相似度检索、上下文构建以及
调用大语言模型生成最终答案。
